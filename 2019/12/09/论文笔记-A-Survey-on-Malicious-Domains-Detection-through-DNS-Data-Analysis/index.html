ABSTRACT

在这篇论文中，作者介绍了一些关于DNS和利用DNS进行恶意活动的背景知识。描述了一个使用DNS数据的恶意域检测技术的通用框架。应用该框架，我们使用多个正交的观点对现有的方法进行分类，即（1）DNS数据的来源及其丰富，（2）数据分析方法，和（3）评估策略和度量。在每一个方面，我们都讨论了研究社区应该解决的重要挑战，以便充分认识到DNS数据分析在对抗利用恶意域的攻击方面的能力。

1.INTRODUCTION

Domain Name System(DNS)域名系统协议是Internet的基本组成部分。它将难以记忆的IP地址映射到容易记忆的域名。通过分析DNS数据检测恶意域名有很多优点：

  • DNS数据只占整个网络流量的一小部分，使得它在即使是覆盖很广的大规模网络中也适合分析。
  • 缓存(caching)，作为协议中不可缺少的一部分，自然得有利于减少要分析的数据量，这使得研究人员可以甚至分析达到顶级域名(Top Level Domains)的DNS流量。
  • DNS流量中包含大量有意义的特性来帮助识别与恶意活动相关联的域名。
  • 这些特性中的许多可以进一步丰富相关信息，例如AS number，域所有人。从而提供更丰富的用于检测的空间。大量的特征和数据使得其很适合许多机器学习算法应用于安全上下文。
  • 尽管加密DNS数据例如DNSCrypt存在，仍然有很大一部分DNS流量仍然未加密，这使得它可用于多种互联网优势点(vantage point)检查。
  • 研究人员能够在攻击的早期，甚至是攻击之前，从DNS数据中留下的一些痕迹揭示攻击。

本文的目的是通过对DNS数据的分析，综述各种检测恶意活动域的方法。作者从多个来源收集文献，建立了一个全面的书目。

  • DNS数据扩充(Data Enrichment)通常的数据扩充需要集成来自不同来源的网络和应用程序数据。比如域注册记录(domain registration records)、自制系统号(autonomous system numbers)和IP托管域的位置信息(geo-location information of IPs hosting domains)。

2.DOMAIN NAME SYSTEM BACKGROUND

2.1 Domain Name System Operation

域名系统是一个分层的分散命名系统(a hierarchical decentralized naming system)，将服务的物理地址(即IP地址)与其逻辑地址（即域名）分离，以便人们可以仅使用其域名连接到服务。
域名被组织为后缀树结构(suffix tree)，称为域命名空间(domain namespace)。最右边的标签命名为顶级域名(Top Level Domain,TLD)。TLD左侧的域是二级域名(Second Level Domain,2LD)。全限定域名(Fully qualified domain name,FQDN），同时带有主机名和域名的名称。资源记录(Resource Records,RRs)，由所有者owner（指示拥有资源记录的DNS域名）、类型type（指定RR类型的编码16位值）、生存时间time-to-live,TTL值（指明其它DNS服务器在期满放弃该记录信息之前对其缓存多长时间）和RDATA字段定义，RDATA字段的内容和语义取决于类型字段的值。本文对以下类型的RR感兴趣：A/AAAA存储IPv4/IPv6主机地址；NS指向存储域信息的权威服务器；MX用于确定邮件的发送位置；PTR记录将主机的IP地址映射到其域名或主机名。
