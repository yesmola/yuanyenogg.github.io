lecture 1: Introduction

为什么人们需要使用大量的计算机协作？

  • parallelism 性能上的需要，很多cpu/gpu/内存等。
  • fault tolerance 一个坏了可以使用另一个
  • physical reasons 物理上的距离
  • security / isolated 通过隔离实现安全性

挑战

  • concurrency 并发性挑战
  • partial failure 部分失败
  • performance 分布式系统的高性能要求

三个基础抽象

  • Storage 存储
  • Communication 通讯
  • Computation 计算

一些主题

  • Implementation 实现：
    RPC，threads，concurrency
  • Performance 性能：
    Scalability - 2x computers -> 2x throughput
  • Fault Tolerance 容错：
    availability 即使有一台计算机出错，也可以使用
    recoverability 可修复性，修复故障后数据可恢复
    两种实现这两个能力的工具：Non-volatile storage / Replication（重要）
  • Consistency 一致性
    Put(k,v)
    Get(k) -> v
    如何保证多个键值对的副本的一致
